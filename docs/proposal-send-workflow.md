# Proposal Send Workflow

## Phase-one MVP

Proposal sending is logged inside Mission Control, while the actual email can be sent manually from the team's normal mailbox.

Flow:

1. Create or edit the proposal.
2. Open the internal proposal preview.
3. Generate the unique proposal link if it does not already exist.
4. Send that link manually by email.
5. Use **Mark sent** / **Log sent** in Mission Control to record the recipient, sent date, sender and note.

Mission Control stores the sent status, sent timestamp, recipient email/name, send method and audit/timeline activity. Full automated email delivery can be added later without changing the proposal record model.
